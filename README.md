# Лабораторна робота №2

## Тема
Документація коду (TypeDoc) і автоматична публікація через GitHub Actions та GitHub Pages

## Мета
Навчитися:
- документувати код стандартними коментарями (TypeDoc);
- автоматично генерувати HTML-документацію в CI/CD;
- публікувати документацію на GitHub Pages після кожного коміту.

## Завдання
1. **Підготовка**
    - Використано код із попередньої лабораторної (калькулятор, логіка `operate` та `calculate`).
    - Створено гілку `feature/docs-ci`.
    - Оновлено README.md, додано розділ Documentation.

2. **Документування коду**
    - Додано коментарі у форматі JSDoc до основних компонентів та функцій.
    - Використано теги: `@param`, `@return`, `@example`.

3. **Генерація документації**
    - Створено конфігураційний файл `typedoc.json`.
    - Перевірено локально генерацію HTML-документації (створено папку `docs/`).

4. **CI/CD через GitHub Actions**
    - Створено `.github/workflows/docs.yml`, який:
        - генерує документацію на кожному push;
        - деплоїть документацію на GitHub Pages.

5. **Публікація**
    - Були увімкнені GitHub Pages у налаштуваннях репозиторію (Source → GitHub Actions).
    - Було перевірено, що після push документація оновлюється за посиланням.

## Юніт-тести
- Тести розташовано у папці `tests/`.
- Використано фреймворк **Jest**.
- Покрито функціональність:
    - компонент `Display`;
    - компонент `Calculator` (обробка кнопок, обчислення виразів);
    - функції `operate` та `calculate` (математичні операції та стан калькулятора);
    - граничні випадки (ділення на нуль, порожні значення).
